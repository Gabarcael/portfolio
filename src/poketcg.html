PokemonTCG

Jeu de cartes numérique développé pendant la SAE 2.01

Description

PokemonTCG est une application web réalisée pour la SAE 2.01. Pendant six semaines, nous avons conçu un deck builder et un simulateur de matchs simplifiés inspirés du jeu de cartes Pokémon. L'accent était mis sur la modélisation métier et la persistance des données. J'ai assuré le rôle de développeur full-stack dans lequel je me suis occupé de la définition de l'architecture, développement des vues principales avec JavaFX.

Mon Rôle

Développeur full-stack et responsable qualité (tests automatisés, revues de code).

Mon Equipe

2 étudiants

Le Livrable

Application déployée et soutenance orale de projet

Galerie

Stack & compétences mobilisées

Compétences techniques

  • Architecture des vues
  • Gestion d'état et composants réutilisables (cartes, deck selon le joueur)
  • Tests pour la logique du jeu
  • Développement avec branches différentes pour chaque features

Soft skills

  • Coordination et communication : Gestion des flux de travail en binôme et synchronisation des versions via Git.
  • Rigueur analytique : Traduction de règles métier complexes (règles du TCG) en algorithmes fonctionnels.
  • Responsabilité et Qualité : Engagement sur la fiabilité du code via la mise en place de tests et de revues de code systématiques.

Outils

Java, JavaFX, GitLab

Retour d'expérience

Points forts

  • Code base modulaire facilitant l'ajout de cartes.
  • Tests automatisés couvrant 85% de la logique métier.
  • Interface claire validée par les professeurs lors de la soutenance.

Difficultés

  • Synchronisation temps réel des états de matchs.
  • Optimisation de l'appel des vues
  • Gestion des collisions de branches Git lors des merges.

Axes d'amélioration

  • Intégration continue pour automatiser les builds et tests.
  • Amélioration de l'UX/UI pour une meilleure expérience utilisateur.
  • Extension des fonctionnalités car toutes n'ont pas pu être implémentéés avant le rendu final
